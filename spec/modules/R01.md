## Goal
FreeSWITCH dialplan + channel-var setup so every bridged call is recorded into a deterministic file path. Includes `recording_follow_transfer=true` so transfers stay captured, and `RECORD_STEREO=true` so agent + customer are on separate channels for analytics.

## Dependencies
- F03 (FS), T03 (agent conf), T04 (originate)

## Blocks (downstream)
- R02 (metadata + S3 upload)
- R03 (playback)
- N07 (transcription)

## Public interface
**Channel variables set on customer leg before bridge:**
- `RECORD_STEREO=true`
- `RECORD_MIN_SEC=2`
- `recording_follow_transfer=true`
- `record_post_process_exec_api=...` (notify our consumer when stop)

**File naming:** `${recordings_dir}/${tenant_id}/${YYYY}/${MM}/${DD}/${campaign_id}_${lead_id}_${uuid}_${start_epoch}.wav`

**Events emitted:** `RECORD_START` and `RECORD_STOP` carry full path + duration.

## Research phase
**MCPs:**
- `mcp__exa__crawling_exa`: FreeSWITCH `record_session` docs, recording follow_transfer notes
- `mcp__exa__web_search_exa`: "FreeSWITCH RECORD_STEREO bridge agent customer separate channels"

**Questions:**
- record_session vs MixMonitor (Asterisk-style)? FS uses `record_session` (mod_dptools).
- Do we record the conference or each leg? **Decision:** record on the customer leg with stereo (agent on right, customer on left); conference-level recording is heavier and adds complexity.
- Where do recordings live in dev (local volume) vs prod (NFS or local then upload to S3 by R02)?

## Verification phase
1. Make a 30s test call; WAV appears at the expected path; stereo (left=customer, right=agent).
2. Transfer customer mid-call; recording continues into the transferred segment.
3. Set campaign mode=NEVER; no recording happens.
4. Set mode=ONDEMAND; agent presses Record (A05); recording starts mid-call.
5. RECORD_STOP event carries final path + duration.

## Test phase
**Integration:** SIPp scenario covers all modes. Bash assertion on file existence + duration.
**Run:** `make test-recording`

## Acceptance criteria
- [ ] All 4 modes work.
- [ ] File path deterministic + parseable.
- [ ] Stereo (agent / customer separated).
- [ ] Survives transfer.
- [ ] RECORD_STOP event emitted with metadata.

## Risks
- **Disk fills.** 8 kHz mono ~ 64 kbps → 28 MB/hour. Stereo doubles. Plan: R02 uploads + deletes locally after upload.
- **`recording_follow_transfer` quirks** in some FS versions. Test on pinned version.
