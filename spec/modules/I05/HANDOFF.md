## Handoff Notes (to be completed post-implementation)

### Mailbox Model

- `voicemail_boxes` table: `id`, `tenant_id`, `ingroup_id`, `user_id`, `did_id`, `greeting_uri`, `max_duration_sec`, `notify_email`, `transcribe`, `active`
- Inbound capture extension naming: `voicemail_{box_id}` in `freeswitch/conf/dialplan/default/75_voicemail_{box_id}.xml`
- Generated by `VoicemailRenderer` (`api/src/services/voicemail/VoicemailRenderer.ts`)

### VM Drop Audio Format

- Accepted upload formats: WAV (PCM), MP3
- Server transcodes to: WAV 8 kHz, mono, 16-bit PCM
- Max duration: 120 seconds
- Max upload size: 10 MB
- Local path pattern: `/var/lib/vici2/vmdrop/{tenant_id}/{asset_id}.wav`
- FreeSWITCH plays via: `uuid_broadcast {uuid} play_and_hangup,{local_path} aleg`

### AMD Integration

- AMD event stream: `events:vici2.call.amd_detected` (Valkey)
- Handler: `dialer/internal/picker/amd_handler.go` — `AMDHandler.handle()`, case `"vmdrop"`
- Config plumbing: `CampaignConfig.AMDAction` + `CampaignConfig.VMDropPath` from M02 config snapshot
- Fallback: if no asset configured → plain `UUIDKill` + audit event `vmdrop_blocked_no_asset`
- Lead status after VM drop: `AVMA`

### Key Configuration Fields

- `Campaign.amdAction` = `vmdrop` (enum)
- `Campaign.vmdropAssetId` = FK → `voicemail_drop_assets.id`
- `Campaign.vmdropRequiresConsent` = `true` (default) — triggers Gate 5 consent check
- `VoicemailBox.notifyEmail` — team email for new-voicemail notification

### E05 / AMD Integration Notes

- I05 extends `AMDHandler` but does not change T03 (AMD detector) or E05 (safe harbor)
- The `"park"` case in `AMDHandler` remains a Phase 3 stub; the new `"vmdrop"` case is Phase 2
- Audit events are written to `events:vici2.audit.vmdrop` stream; API worker consumes and writes to `audit_log`
