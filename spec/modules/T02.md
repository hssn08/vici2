## Goal
Admin UI / API for adding/editing/deleting SIP carriers (Twilio, RingCentral, generic BYOC); writes Sofia gateway XML to `freeswitch/conf/sip_profiles/external/<name>.xml`; triggers `sofia profile external rescan` via ESL; live status (REGED/UNREG) surfaced to admin UI.

## Dependencies
- F02 (`carriers` table)
- F03 (FS with external profile)
- F05 (RBAC: only admin/superadmin)
- T01 (ESL for rescan)

## Blocks (downstream)
- T04 (originate needs a carrier name)
- M04 (admin UI)
- I02 (DID inbound routing)
- X04 (number pool)

## Public interface
**REST:**
- `GET /api/admin/carriers` — list with live registration state
- `POST /api/admin/carriers` — create
- `PATCH /api/admin/carriers/:id` — update (triggers re-render + rescan)
- `DELETE /api/admin/carriers/:id` — soft delete (file removed; rescan)
- `POST /api/admin/carriers/:id/test` — sends OPTIONS via ESL; returns response code + latency
- `GET /api/admin/carriers/:id/status` — live REGED/UNREG/FAIL state

**Templates:**
- `freeswitch/conf/sip_profiles/external/<carrier_name>.xml` written from a Handlebars/EJS template per carrier `kind`.

**Pseudocode:**
```typescript
async function upsertCarrier(c: Carrier) {
  const xml = render(c.kind, c);
  await writeFileAtomic(`/freeswitch/conf/sip_profiles/external/${c.name}.xml`, xml);
  const result = await esl.api('sofia profile external rescan');
  if (result.includes('+ERR')) {
    // revert
    await restoreBackup(...);
    throw new AppError('CARRIER_RELOAD_FAILED', 500, { result });
  }
  await audit({ event: 'carrier.upserted', carrier_id: c.id });
}
```

## Verification phase
1. Add a Twilio carrier via API (test creds); XML appears, `sofia status` shows new gateway.
2. Add a RingCentral carrier with `register=true`; within 30s `REGED` status visible.
3. `POST /api/admin/carriers/:id/test` returns OPTIONS reply.
4. Edit caller-ID → XML re-rendered, rescan triggered, no live calls dropped.
5. Delete carrier → XML removed; `sofia status` no longer lists.
6. Concurrent updates serialized (no race deleting half-written XML).
7. Status cache reflects FS reality within 60s of any change.

## Acceptance criteria
- [ ] All 4 `kind` templates render valid XML (xmllint passes).
- [ ] CRUD endpoints work + RBAC enforced.
- [ ] Live status visible in <60s of change.
- [ ] Rescan failure rolls back file change.
- [ ] Encrypted-at-rest passwords; never returned in API responses (replace with `***`).
- [ ] Audit log for create/update/delete.
- [ ] Soft delete removes XML file.
- [ ] OPTIONS test endpoint works.

## Risks
- **Carrier-specific quirks** (e.g., RingCentral needs custom `register-proxy`). Each `kind` gets its own template; document quirks.
- **XML injection via carrier name/password fields.** Handlebars escapes; tests verify with malicious payload.
- **Race between two admins editing same carrier.** Use optimistic concurrency (`version` column).
