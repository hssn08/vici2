## Goal
A central guard called from every dial path (manual, auto, transfer 3rd party) that verifies `localTime(phone) ∈ campaign.callTimes ± stateOverrides`. Returns structured deny reason. Used by E01 (hopper filler) and T04 (final pre-originate).

## Dependencies
- D03 (resolver), F02 (`call_times`)

## Blocks
- T04, A04, A07, E01

## Public interface
```typescript
function assertCallWindow(opts: { phoneE164, campaignId, when?: Date }): Promise<void> | throws AppError('OUTSIDE_CALL_WINDOW', { localHour, allowed })
```

## Verification phase
1. Lead in CA at 3 AM Pacific → throws.
2. Lead in NY at 10 AM Eastern → passes.
3. State override stricter than campaign window → stricter wins.
4. Unknown TZ → optionally pass with warning (configurable per campaign).

## Test phase
**Unit:** edge cases around DST transitions, unknown TZ, state overrides.

## Acceptance criteria
- [ ] Single function used by all dial paths.
- [ ] Most-restrictive rule wins (state override).
- [ ] Configurable behavior on unknown TZ (default: deny).
- [ ] Audit metric per deny.

## Risks
- **Stale `Date.now()` in long-running dials** — recheck at originate time, not just at hopper insert (T04 calls it again).
