## Goal
On CHANNEL_ANSWER, atomically pick a READY agent for this campaign by `next_agent_call` strategy: longest_wait, random, fewest_calls, rank. Reserve them (remove from READY ZSET) and return. If none available within budget (configurable, default 1.5s) → return null → caller drops with safe-harbor.

## Dependencies
- F04 (Redis ZSETs), F02 (campaigns.next_agent_call)

## Blocks
- T04 / answer handler uses this; E05 records drops if no agent.

## Public interface
```go
func PickAgent(ctx, tenantID, campaignID, strategy) (userID int64, ok bool, err error)
```

## Verification phase
1. With 5 READY agents, picker returns longest-waiting one each time.
2. Random strategy gives statistical fairness over 1000 picks.
3. Picker atomic: 100 concurrent picks return 100 distinct agents (until pool empty).
4. Budget exhausted → returns ok=false within 1.5s.

## Test phase
**Unit:** strategy correctness.
**Integration:** concurrent pick stress.

## Acceptance criteria
- [ ] All 4 strategies.
- [ ] Atomic pick (no double-assign).
- [ ] Budget enforcement.
- [ ] Releases agent back to READY on any failure path.

## Risks
