## Goal
Once a call is bridged, the agent screen shows lead info (name, phone, address, prior call history, comments), embedded script tab, embedded webform iframe (optional), call timer, and the in-call control bar (Hangup, Hold, Mute, DTMF, Record toggle for ONDEMAND).

## Dependencies
- A02, A03, A04, D01, R01 (recording control)

## Blocks
- A06 (dispo picker), A07 (transfer), A08 (callback), A09 (pause)

## Public interface
**Layout components:**
- `<LeadInfoPanel lead />`
- `<CallControlBar callUuid />`
- `<ScriptTab campaignId lead />`
- `<WebformIframe url lead callUuid />`
- `<HistoryTab leadId />`
- `<NotesTab lead />`

## Verification phase
1. Bridge a call; lead info panel shows fields; timer ticks.
2. Click Hold; remote MoH; Unhold resumes.
3. Click Mute; remote silence; Unmute resumes.
4. DTMF: open keypad, press digits; remote receives.
5. Recording toggle (ONDEMAND mode): start/stop visibly.
6. Script tab shows campaign script with `{{lead.first_name}}` populated.
7. Webform iframe loads campaign URL; postMessage from form updates lead via API.
8. History tab shows prior call_log entries for this lead.

## Test phase
**Unit:** template substitution; postMessage handler.
**E2E:** Playwright runs through every control.
**Run:** `cd web && npm run test:e2e -- call-panel`

## Acceptance criteria
- [ ] All controls work via SIP.js primitives.
- [ ] Lead fields render with template substitution.
- [ ] Webform iframe sandboxed; postMessage validated.
- [ ] History shows past 10 calls.
- [ ] Timer accurate to 1s.
- [ ] Mobile-readable at 768px width.

## Risks
- **iframe XSS via webform URL** — strict origin allowlist; sandbox attributes.
- **postMessage spoofing** — verify event.origin matches campaign.webform_url origin.
