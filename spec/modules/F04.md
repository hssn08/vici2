## Goal
Define the canonical Redis key namespace, data types, and TTL conventions for all live state in vici2. Ship a thin helper library in both Go (`dialer/internal/redis`) and TS (`api/src/lib/redis`) that enforces the conventions.

## Public interface
**Key namespace (canonical):**
```
t:{tenant_id}:agent:{user_id}                 HASH    {status, campaign_id, lead_id, call_uuid, last_change_at, pause_code, ingroups[], server, sip_state}
t:{tenant_id}:agents:by_status:{STATUS}       ZSET    score=last_change_at; members=user_id
t:{tenant_id}:agents:by_campaign:{cid}:READY  ZSET    score=last_change_at; members=user_id
t:{tenant_id}:campaign:{cid}:dial_level       STRING  decimal as text
t:{tenant_id}:campaign:{cid}:drop_window      STREAM  {answered, dropped, ts}; trim 30d
t:{tenant_id}:campaign:{cid}:hopper           ZSET    score=priority desc + entry_ts; members=lead_id
t:{tenant_id}:campaign:{cid}:active_calls     SET     active originated UUIDs
t:{tenant_id}:hopper:lock:{cid}:{lead_id}     STRING  TTL 30s; claim by dialer instance
t:{tenant_id}:call:{uuid}                     HASH    {lead_id, campaign_id, agent_id, started_at, state, carrier_id}
t:{tenant_id}:call:active                     SET     all currently-active call UUIDs (any direction)
t:{tenant_id}:broadcast:agent:{user_id}       PUBSUB  per-agent state push channel
t:{tenant_id}:broadcast:campaign:{cid}        PUBSUB  campaign-wide events (drop%, ready count)
t:{tenant_id}:broadcast:wallboard             PUBSUB  supervisor wallboard fan-out
t:{tenant_id}:janitor:lock                    STRING  TTL 60s; only one janitor instance runs at a time
t:{tenant_id}:dialer:tick:{cid}               STRING  TTL 1s; dedup pacing ticks across dialer instances
t:{tenant_id}:dnc:cache:{phone_e164}          STRING  TTL 1h; "1" if DNC, else absent (negative cache ok)
events:vici2.call.answered                    STREAM  consumer-group fan-out (cross-tenant tag in payload)
events:vici2.call.bridged                     STREAM  same
events:vici2.call.ended                       STREAM
events:vici2.call.dropped                     STREAM
events:vici2.agent.state_changed              STREAM
```

**Helper library responsibilities:**
- Build keys via typed builders so no agent ever hard-codes a key string.
- Enforce TTLs on transient keys.
- Wrap `XADD MAXLEN ~ N` / time-based trim on streams.
- Provide atomic operations (e.g., `claim_lead_from_hopper`) as a single Lua-scripted call.
- Provide test helpers (`redis-mock` for unit tests, real Redis for integration).

**Questions to answer:**
- Should we use `XADD MAXLEN ~ N` (approximate trim, fast) or time-based trim (precise, slower)? **Recommendation:** approximate by count for hot streams; periodic time-trim cron for the 30-day drop_window.
- Should agent state be a HASH or a JSON STRING? **Recommendation:** HASH — atomic field updates without read-modify-write.
- How to make `ZPOPMIN` + write a "claim" key atomic? **Recommendation:** Lua script.
- How does Redis Cluster route streams across nodes? (Hash-tagged key names in `{}`.)
- Memory budget for hopper at 50 campaigns × 500 leads each? (~25K entries × 64B = 1.6MB; trivial.)
- TTL hygiene: which keys could leak forever if a service crashes? Audit list.

**Pseudocode — atomic hopper claim (Lua):**
```lua
-- KEYS[1] = hopper sorted set
-- KEYS[2] = lock prefix (we'll append lead_id)
-- ARGV[1] = lock TTL seconds
-- Returns: lead_id or nil
local popped = redis.call('ZPOPMIN', KEYS[1], 1)
if #popped == 0 then return nil end
local lead_id = popped[1]
redis.call('SETEX', KEYS[2] .. lead_id, ARGV[1], '1')
return lead_id
```

**Pseudocode — record drop (Lua):**
```lua
-- KEYS[1] = drop_window stream
-- KEYS[2] = events stream
-- ARGV[1] = answered (0/1), ARGV[2] = dropped (0/1)
-- ARGV[3] = ts, ARGV[4] = call_uuid, ARGV[5] = campaign_id, ARGV[6] = tenant_id
redis.call('XADD', KEYS[1], 'MAXLEN', '~', '500000', '*',
  'answered', ARGV[1], 'dropped', ARGV[2], 'ts', ARGV[3])
redis.call('XADD', KEYS[2], 'MAXLEN', '~', '1000000', '*',
  'tenant_id', ARGV[6], 'campaign_id', ARGV[5],
  'call_uuid', ARGV[4], 'ts', ARGV[3])
return 'OK'
```

## Risks
- **Type drift between Go and TS** — mitigate with shared JSON-Schema definitions in `shared/events/` + code-gen (Phase 4 if needed; Phase 1 just discipline).
- **Lua script eviction in dev Redis restarts** — load on boot via `SCRIPT LOAD` and detect `NOSCRIPT` errors with re-load.
- **Single-node Redis is a SPOF.** Plan: document Redis Sentinel / Cluster path (in HANDOFF.md), defer to Phase 3.5.
