## Goal
Manage pools of DIDs per campaign; rotate caller-ID across pool to prevent any single number from being labeled "Spam Likely". Track per-number health (answer rate, complaint rate) and quarantine bad numbers.

## Dependencies
- T02 (DIDs), E02 (originate caller-ID selection)

## Public interface
**REST:** `/api/admin/number-pools` CRUD; assign DIDs.
**Service:** `pickCallerId(campaignId, leadE164) → e164`.

## Plan phase
**Deliverable:** `spec/modules/X04/PLAN.md`: pool data model; rotation algo (round-robin with health weighting); quarantine criteria.

## Verification phase
1. Pool of 10 DIDs assigned to campaign; outbound calls cycle through.
2. DID with answer-rate < 5% over 1000 calls auto-quarantined.
3. Admin manually unquarantine.
4. Per-DID metric visible.

## Test phase
