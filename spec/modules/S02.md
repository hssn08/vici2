## Goal
Supervisor selects an active call → originates a leg from their browser via SIP.js into FS that runs `eavesdrop:<uuid>`. DTMF toggles modes: 1=listen, 2=whisper-to-agent (agent hears, customer doesn't), 3=barge-3way.

## Dependencies
- T01 (ESL), A02 (supervisor's softphone)

## Public interface
**REST:** `POST /api/sup/eavesdrop` `{ call_uuid, mode: 'listen'|'whisper'|'barge' }`
**UI:** Wallboard agent tile → "Listen" / "Whisper" / "Barge" buttons.

## Verification phase
1. Supervisor clicks "Listen" on agent A's call → supervisor hears bidirectional silently; agent + customer continue.
2. DTMF '2' → whisper mode; agent hears supervisor; customer doesn't.
3. DTMF '3' → barge; all three on the line.
4. Audit log per eavesdrop session (compliance: who listened to what when).
