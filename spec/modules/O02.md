## Goal
Nightly MySQL backups (logical via mysqldump or physical via Percona XtraBackup), Redis RDB snapshots, FreeSWITCH config (already in git, but lazy backup tarball), recordings already in S3 (cross-region replication on for prod). Documented restore procedure.

## Dependencies
- F02

## Public interface
- Cron job runs nightly at 02:00
- Backups upload to S3 with retention policy (30 daily + 12 monthly)
- `make restore-from-backup BACKUP=<id>` script

## Verification phase
1. Run backup; verify S3 has artifacts.
2. Restore into a test environment; data correct.
3. RTO < 60 min; RPO < 24 h documented and met.

## Acceptance criteria
- [ ] Nightly backups; verified upload.
- [ ] Tested restoration.
- [ ] Retention policy enforced.
- [ ] Runbook clear.
- [ ] Encryption at rest in S3.

## Risks
