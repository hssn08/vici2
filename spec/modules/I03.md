## Goal
Admin can compose IVRs visually (drag-drop tree of: prompt → keypress → action). Compiled to FS dialplan extensions served via mod_xml_curl. Actions: ingroup, transfer, voicemail, sub-IVR, hangup, custom audio.

## Dependencies
- I01, I02, F03 (mod_xml_curl)

## Public interface
**REST:** `/api/admin/ivrs` CRUD.
**UI:** `(admin)/ivrs` with React Flow tree builder.

## Verification phase
1. Build a 3-level IVR; save; assign to a DID; inbound caller traverses correctly.
2. Audio prompts uploaded + assigned per node.
3. Invalid keypress → re-prompt up to N times → action.
4. Sub-IVR works (cycles must be detected + rejected).
5. Cyclic IVR detection on save.

## Test phase
**Unit:** compiler; cycle detection.
**E2E:** end-to-end with SIPp DTMF input.

## Acceptance criteria
- [ ] Visual builder.
- [ ] Compiled FS XML correct.
- [ ] Audio upload integrated.
- [ ] Cycle detection.
- [ ] Per-IVR analytics: keypress distribution, drop-off rate.

## Risks
- **Cycle detection** must be exhaustive; cycles can come from sub-IVR refs.
