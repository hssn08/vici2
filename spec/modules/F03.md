## Goal
Produce a FreeSWITCH 1.10 deployment (Dockerized for dev, host-ready for prod) configured with: ESL enabled, Sofia internal profile (UDP + WSS for browsers), Sofia external profile (carrier gateways live in subdir), default conference profile tuned for agent-conf-per-user, mod_callcenter loaded, mod_avmd loaded, baseline ACL & security.

- T02 (carrier gateways depend on `external/` directory)
- T03 (agent conference dialplan needs base config)
- T04 (originate needs gateways + ESL)
- A02 (browser SIP.js needs WSS port reachable)

## Public interface
- FS reachable on `127.0.0.1:5060/udp` (SIP), `127.0.0.1:7443/tcp` (WSS), `127.0.0.1:8021/tcp` (ESL with password from env).
- ACL `domestic` allowing localhost + Docker subnet for ESL.
- Static dialplan contexts `default` and `public` exist with safe defaults (untrusted public traffic dropped).
- Conference profile `default` configured (no MOH on idle, mute-detect off, optimal for agent-per-conf).
- WSS cert (self-signed for dev) loaded.
- Module load list explicitly enumerated in `modules.conf.xml`.
- `fs_cli` works locally; remote ESL connect documented.

**Key non-obvious decisions to bake in:**
1. **ulimit nofile = 1048576** in Dockerfile + Compose (`ulimits:` block). Mitigates ~1800 session thread wall partially.
2. **`max-sessions = 5000` and `sessions-per-second = 100`** in switch.conf.xml — explicit upper bound for safety.
3. **Internal profile binds 5060 UDP/TCP for SIP, plus 7443 for WSS** (single profile handles both).
4. **External profile binds 5080** — carriers register here; agent SIP cannot reach.
5. **ESL `listen-ip` = `0.0.0.0` in dev**, `127.0.0.1` in prod. ACL prevents internet access.
6. **`event_socket.conf.xml` password is `ENV-driven`** via vars.xml `${ESL_PASSWORD}`.
7. **Conference default profile**:
   - `enter-sound`/`exit-sound` = silent (we don't want sounds when calls join agent's conf)
   - `wait-mod` = false (no waiting for moderator)
   - `caller-id-name`/`caller-id-number` set per-channel via uuid_setvar
   - `comfort-noise`, `interval=20`, `rate=8000` (PCMU friendly)
8. **No `mod_xml_curl` in Phase 1** — static dialplans only. Add in I03.
9. **mod_avmd loaded but disabled-by-default** in dialplan; opted in per call.

## Verification phase
**Manual checks:**
1. `docker compose up freeswitch` → `fs_cli` connects via `fs_cli -H 127.0.0.1 -p $ESL_PASSWORD`.
2. `sofia status` shows `internal` and `external` profiles RUNNING.
3. `sofia status profile internal` shows WSS binding on 7443.
4. `module list` shows: mod_sofia, mod_event_socket, mod_dptools, mod_conference, mod_callcenter, mod_avmd, mod_db, mod_loopback, mod_local_stream, mod_native_file, mod_sndfile, mod_tone_stream, mod_dialplan_xml. **No** mod_xml_cdr, mod_xml_curl (Phase 3), mod_voicemail (Phase 3), mod_v8/lua unless needed.
5. `acl list` shows our ACLs.
6. From host: `openssl s_client -connect localhost:7443` shows valid (self-signed) cert.
7. SIPp test as a client registers against internal profile.
8. Healthcheck script returns 0 when FS is healthy, non-zero on failure.
9. `docker stats freeswitch` shows ~50MB RAM idle.
10. Container restart cleanly preserves no state (we don't want stuck sessions on dev restart).

## Risks
- **WSS cert chain on dev laptops** — browsers refuse self-signed certs without trusting CA. Provide `mkcert -install` script in `gen-dev-cert.sh`.
- **Mac Docker host networking** — RTP won't work without host-network. Document; suggest Linux VM or alternate "media-relay-via-rtpengine" config (Phase 2 X01).
- **`mod_callcenter` config schema** — empty config in Phase 1 is fine; I01 fills in queues.
- **`switch_thread_create()` wall** — even with ulimits, the design constraint stands. Document expected ceiling of ~1800 sessions per FS instance.
