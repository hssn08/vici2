## Goal
Bootstrap a single-command (`make dev`) local development environment: MySQL 8, Redis 7, FreeSWITCH 1.10, and stub services (api, dialer, web, workers) all running together via Docker Compose.

## Dependencies
None.

## Blocks (downstream)
**Everything.** No other module can begin without F01.

## Public interface
- `make dev` brings up the full stack on localhost.
- `make down` tears it all down cleanly.
- `make seed` populates demo data (after F02 done).
- `make test` runs all unit tests across services.
- `docker-compose.yml` exposes:
  - `mysql:3306` (db `vici2`, user `vici2`, password from env)
  - `redis:6379`
  - `freeswitch:5060/udp`, `7443/tcp` (WSS), `8021/tcp` (ESL)
  - `api:3000` (HTTP), `9101` (metrics)
  - `dialer:9102` (metrics; gRPC on `:7000`)
  - `workers:9103` (metrics)
  - `web:4000` (HTTP)
- `.env.example` documents every env var.
- `README.md` contains a 5-minute quick-start.

**Key non-obvious decisions to bake in:**
1. **FreeSWITCH in `network_mode: host`** for dev — RTP needs it. Document Mac/Windows caveat (Docker Desktop doesn't truly host-network on Mac; needs alternate config).
2. **Workspace strategy:** npm workspaces for `api/web/workers` (TS); separate Go module for `dialer`. Shared TS types live in `shared/types/` (compiled but not published).
3. **Hot reload:** Node services use `tsx watch`; Go uses `air`; Next uses built-in.
4. **Healthchecks:** every service has a `/health` endpoint; Compose `healthcheck:` blocks ensure ordered startup.
5. **Logging:** every service writes JSON to stdout; Compose collects.
6. **Single-process services in dev** so `docker compose logs -f api` is useful.

## Verification phase
**Manual checks:**
1. `git clone <repo> && cp .env.example .env && make dev` — comes up clean from cold.
2. `curl http://localhost:3000/health` → `{"status":"ok"}`
3. `curl http://localhost:9102/metrics` → Prometheus format from dialer
4. `mysql -h 127.0.0.1 -uvici2 -p$VICI2_DB_PASSWORD vici2 -e 'SELECT 1'`
5. `redis-cli -h 127.0.0.1 PING` → `PONG`
6. `fs_cli -H 127.0.0.1 -p ClueCon` connects (use dev password from .env)
7. Open `http://localhost:4000` — Next.js "hello" renders.
8. Edit `api/src/server.ts`, save — auto-reloads.
9. `make down && docker volume ls | grep vici2` — no orphan volumes left.
10. `make reset` — wipes data volumes, returns to clean state.

## Acceptance criteria
- [ ] `make dev` from a clean clone brings up all 5 services in <120 seconds.
- [ ] All 10 verification steps pass.
- [ ] All Dockerfiles produce images <500 MB each (FS may be larger; document if so).
- [ ] `.env.example` has zero secrets, every var documented with comment.
- [ ] README has a quick-start a new dev can follow without questions.
- [ ] CI workflow file present (even if O04 will rewrite it).
- [ ] `make down` leaves the system clean (no dangling containers, no orphan volumes unless `--volumes` used).

## Risks
- **Mac users may struggle with FS host networking.** Document the workaround (custom bridge + extra port mapping, accept that WebRTC won't work without further config) and note the goal is Linux-first dev environments.
- **FreeSWITCH startup is slow (~30s).** Make Compose dependency ordering tolerant.
