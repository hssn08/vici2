## Goal
A single typed `Originate(opts)` function in the dialer that issues `bgapi originate` via the carrier's gateway, parks the answered customer leg, then bridges into the target agent's conference (or holds for the dialer engine to assign). Writes a `call_log` row at originate time and updates it through the call lifecycle via T01 events.

## Dependencies
- T01 (ESL Originate)
- T02 (carrier name lookup)
- T03 (agent conference name)
- F02 (`call_log` table)
- F04 (Redis call:active set, call:{uuid} hash)

## Blocks (downstream)
- A04 (manual dial)
- E02 (auto-dial pacing)
- I04 (closer transfer)
- N01 (external API click-to-dial)

## Public interface
**Go:**
```go
type OriginateOpts struct {
  TenantID    int64
  CampaignID  string
  LeadID      int64
  PhoneE164   string
  CarrierID   int64           // resolved to gateway name
  CallerID    string          // E.164
  TimeoutSec  int             // default 22
  AssignTo    *AgentTarget    // nil = park; non-nil = bridge to agent conf on answer
  Recording   RecordingMode   // OFF / ON / FORCED
  Headers     map[string]string  // SIP X-headers for downstream context
}
type AgentTarget struct { UserID int64 }

func (e *Engine) Originate(ctx context.Context, opts OriginateOpts) (callUUID string, err error)
```

**Side effects:**
- `call_log` row inserted (status=`INPROG`, direction=`out`).
- `call:active` set updated with the new UUID.
- `call:{uuid}` hash populated.
- ESL bgapi originate fired.
- BACKGROUND_JOB result tracked; on failure, `call_log` updated to FAILED with hangup_cause.

**Questions to answer:**
- `&park()` (we control via uuid_transfer on CHANNEL_ANSWER) vs `execute_on_answer=transfer:conf_X XML default` (FS auto-transfers). **Recommendation:** `&park()` + we explicitly bridge — gives us a window to AMD, TZ-recheck, drop-rate-check before agent connect.
- How to inject custom SIP headers (carrier-specific): `sip_h_X-CustomHeader=...` channel variable.
- How to set caller ID and have it survive carrier rewriting (Twilio): `origination_caller_id_number` + `effective_caller_id_number` + `sip_from_user`.
- How to know when the customer truly answered (vs early media): listen for `CHANNEL_ANSWER`, **not** `CHANNEL_PROGRESS_MEDIA`.
- Hangup cause mapping: `NO_ANSWER`, `USER_BUSY`, `NORMAL_CLEARING`, `CALL_REJECTED` etc. → our `dispo` mapping (NA, B, A=answered).

**Pseudocode — originate string:**
```
originate {
  origination_uuid=${call_uuid},
  origination_caller_id_number=${cid_e164},
  origination_caller_id_name=${cid_name},
  effective_caller_id_number=${cid_e164},
  ignore_early_media=true,
  call_timeout=${timeout_sec},
  hangup_after_bridge=true,
  campaign_id=${campaign_id},
  lead_id=${lead_id},
  tenant_id=${tenant_id},
  carrier_id=${carrier_id},
  vici2_role=customer_leg,
  sip_h_X-Vici2-Lead=${lead_id},
  sip_h_X-Vici2-Campaign=${campaign_id}
}sofia/gateway/${carrier_name}/${phone_e164} &park()
```

**Pseudocode — answer handler (lives in api consumer):**
```typescript
on(CHANNEL_ANSWER, evt => {
  const uuid = evt.uuid;
  const ctx = await loadCallContext(uuid);   // reads call:{uuid}
  // compliance final-check (some seconds may have passed)
  if (!withinCallWindow(ctx.tz_offset_min, ctx.state)) return await killCall(uuid, 'TZ_GATE');
  if (await isDnc(ctx.phone_e164)) return await killCall(uuid, 'DNC_GATE');
  // assign agent
  const agentId = ctx.assignedAgentId
    ?? await pickAgent(ctx.tenant_id, ctx.campaign_id);  // E04 in Phase 2; for Phase 1 manual dial assigns explicitly
  if (!agentId) {
    // ABANDON — drop with safe-harbor
    await playSafeHarborAndHangup(uuid, ctx.campaign_id);
    await recordDrop(ctx);
    return;
  }
  // bridge into agent's conf
  await esl.api(`uuid_transfer ${uuid} conference_${agentId} XML default`);
});
```

## Verification phase
1. With a SIPp scenario simulating Twilio: originate to `+12125550100`. Customer SIPp answers; call is parked. Manually run `uuid_transfer <uuid> conference_42 XML default` — customer joins agent's conf; bidirectional audio works.
2. Originate with `AssignTo={UserID:42}`: on answer, customer auto-bridged into agent 42's conf.
3. Originate to a "busy" number (SIPp returns 486) → call_log row updated with `hangup_cause=USER_BUSY`, `status=B` (per status mapping).
4. Originate to a "no answer" → after `dial_timeout_sec`, hangup_cause=NO_ANSWER, status=NA.
5. Carrier returns 503 → status=NETWORK, alert metric incremented.
6. SIP X-headers present in INVITE on the wire (verify via `sofia tracelevel info` + tcpdump).
7. 50 concurrent originates fire without errors; all end up in correct call_log states.

## Risks
- **Race between CHANNEL_ANSWER and assignTo agent already mid-disconnect.** Picker (E04) must atomically claim.
- **Carrier early-media spuriously triggering "answer" detection.** `ignore_early_media=true` is mandatory.
- **Custom X-headers stripped by some carriers.** Document per carrier; fall back to channel-vars on bridge.
