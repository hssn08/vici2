## Goal
Agents schedule callbacks (agent-only or anyone) at a future time. A worker fires due callbacks: "agent-only" routes to that specific agent when ready; "anyone" inserts into hopper (Phase 2) or surfaces in admin UI (Phase 1).

## Dependencies

- D01 (lead linkage)

## Blocks (downstream)
- A08 (callback UI)
- E01 (hopper filler reads anyone-callbacks)
- M02 (campaign callback toggle)

## Public interface
**REST:**
- `POST /api/agent/callbacks` — `{ lead_id, callback_at, agent_only, comments }`
- `GET /api/agent/callbacks/mine?status=PENDING` — agent's pending callbacks
- `POST /api/agent/callbacks/:id/snooze` — push to later time
- `POST /api/agent/callbacks/:id/cancel` — mark DEAD
- `GET /api/admin/callbacks?campaignId=&status=` — all
- `POST /api/admin/callbacks/:id/reassign` — change owner

**Worker:**
- Every minute: `SELECT * FROM callbacks WHERE status='PENDING' AND callback_at <= NOW()`
- For agent-only: set `status='LIVE'`; emit `vici2.callback.due.agent` event → API delivers to agent UI as a notification.
- For anyone: set `status='LIVE'`; in Phase 2, E01 includes these in hopper. In Phase 1, surfaces in admin UI.

## Verification phase
1. Schedule callback in 2 min for self; worker fires at T+2; agent UI shows notification; clicking dials lead.
2. Schedule callback (anyone) for tomorrow; admin sees in queue.
3. Snooze updates callback_at; fires later.
4. Cancel marks DEAD; not fired.
5. Worker is idempotent (re-running on same minute doesn't double-fire).

## Test phase
**Unit:** state transitions; snooze validation.
**Integration:** worker actually picks due callbacks; agent-only routing works.
**Run:** `npm test -- callbacks`

## Acceptance criteria
- [ ] Callback CRUD endpoints work.

- [ ] Agent-only routes to specified agent only.
- [ ] Anyone callbacks visible to admin (Phase 1) and feed hopper (Phase 2 hookup).
- [ ] Snooze + cancel work.
- [ ] Audit log on schedule/snooze/cancel/fire.

## Risks
- **Callback overrun**: 1000 callbacks all due at 9am Monday — worker must batch.
- **Time-zone confusion** — store UTC; render in agent's tz.
