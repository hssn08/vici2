# Module A06 — Disposition + Hotkeys + Wrapup

| Field | Value |
|---|---|
| Track | Agent UI |
| Phase | 1 |
| Effort | 3–4 days |
| Owner agent type | frontend + backend-node |
| Status | NOT_STARTED |

## Goal
After Hangup, present disposition picker with campaign-specific statuses + hotkeys; on submit, write status + agent_log + start wrap-up timer; then return to READY. If hotkeys enabled, single keypress dispositions and (optionally) auto-dials next.

## Dependencies
- A05, D01 (setStatus), D04 (statuses + hotkeys)

## Blocks
- A04 (auto-dial-next on hotkey if manual_progressive setting)
- M07 (admin status admin)

## Public interface
**REST:**
- `POST /api/agent/dispo` — `{ call_uuid, status, comments?, callback?: { at, agent_only } }`

**UI:**
- `<DispositionPicker statuses hotkeys onSubmit />`
- `<WrapUpTimer seconds onTimeout />`
- Global keyboard handler bound only when picker is open.

## Research phase
- Hotkey conflicts with browser shortcuts (avoid Ctrl/Cmd+key).
- A11y: keyboard nav for picker.

## Verification phase
1. End a call; dispo picker appears.
2. Click SALE; lead.status='SALE'; agent_log entry written; wrap timer starts; flips to READY after wrapup_seconds.
3. With hotkeys ON: press '1' = SALE; auto-submits.
4. CALLBK selection prompts callback time; on save, callback row created.
5. Pause picker if comments required; submit blocked without comments.
6. Cannot dispo if no active call (button disabled).

## Test phase
**Unit:** validators; hotkey-to-status mapping.
**E2E:** Playwright runs through hotkey + manual paths.
**Run:** `npm test -- dispo`

## Acceptance criteria
- [ ] Dispositions write status + agent_log atomically.
- [ ] Hotkeys: 0–9 + Enter; configurable per campaign.
- [ ] Wrapup timer respects campaign setting.
- [ ] CALLBK creates a callback row.
- [ ] DNC dispo writes to internal DNC.
- [ ] Required-comments enforced.
- [ ] After wrapup, agent state → READY automatically.

## Risks
- **Lost dispo if browser closes** — auto-save draft to localStorage; recover on next login.
