# Module I05 — Voicemail Capture + Drop

| Field | Value |
|---|---|
| Track | Inbound |
| Phase | 3 |
| Effort | 3–4 days |
| Owner agent type | telephony + backend-node |
| Status | NOT_STARTED |

## Goal
Inbound caller routed to voicemail records a message. Outbound campaign with `amd_action=vmdrop` plays a pre-recorded voicemail-drop audio when AMD detects a machine.

## Dependencies
- F03 (mod_voicemail or our own simple recorder), I01/I02/I03 (route to vm), R01 (recording infra)

## Public interface
**Mailboxes:** per-DID or per-ingroup voicemail target.
**REST:** `/api/admin/voicemails` browse; `/api/admin/vmdrops` upload audio per campaign.

## Verification phase
1. Inbound to a DID with vm target → caller hears greeting + records; recording stored; admin sees in inbox.
2. Outbound campaign with vmdrop: AMD detects machine → vmdrop audio plays → hangup; lead.status='AVMA'.
3. Per-campaign vmdrop audio configurable.
