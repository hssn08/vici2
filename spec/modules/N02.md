## Goal
Per-tenant subscribers to events (call.started, call.ended, dispo.set, transfer.complete, etc.). HMAC-signed POST to subscriber URL with retries + DLQ.

## Dependencies
- T01 (events), C03 (audit log feed), F04 (Redis streams)

## Blocks
- N03, N04 (CRM adapters subscribe via this)

## Public interface
**REST:** `/api/admin/webhooks` CRUD.
**Behavior:** worker reads `events:vici2.*` streams; matches subscriber filters; POSTs with `X-Vici2-Signature: <hmac>`; retry up to 5x exponential.

## Plan phase
**Deliverable:** `spec/modules/N02/PLAN.md`: subscriber model; payload schemas; signature scheme; retry policy; DLQ.

## Verification phase
1. Subscribe to `call.ended` for a tenant; trigger a call; webhook fires.
2. Subscriber returns 500 → retried with backoff up to 5x; then DLQ.
3. Subscriber URL invalid → DLQ immediately.
4. HMAC verifiable on receiver side using shared secret.
5. Webhook logs visible in admin UI.
