## Goal
A robust ESL client in the Go dialer (issues `bgapi originate`, `uuid_*`, `conference` commands) and a parallel ESL listener in the Node API (subscribes to channel events for WebSocket fan-out + DB writes). Both auto-reconnect, both reconcile state on (re)connect.

## Public interface
**Go (`dialer/internal/esl`):**
```go
type Client interface {
  BgAPI(ctx, cmd, args string) (jobUUID string, err error)   // returns immediately; result via event
  API(ctx, cmd, args string) (body string, err error)        // sync call
  Originate(ctx, opts OriginateOpts) (callUUID string, err error)
  UUIDTransfer(ctx, uuid, dest string) error
  UUIDBridge(ctx, leg1, leg2 string) error
  UUIDKill(ctx, uuid, cause string) error
  ConferenceList(ctx, name string) ([]Member, error)
  ConferenceKick(ctx, name, memberID string) error
  Reload(ctx, what string) error                              // "xml", "sofia profile X rescan"
}
```
**Node (`api/src/esl`):**
- `EslListener` connects, subscribes `events plain ALL`, publishes parsed events to:
  - Redis stream `events:vici2.call.*`
  - Redis pub/sub `t:{tid}:broadcast:agent:{user_id}` for relevant channel events
  - Direct DB writes for `call_log` updates (started_at, answered_at, ended_at, hangup_cause)
- Subscribed event filters: `CHANNEL_CREATE`, `CHANNEL_PROGRESS`, `CHANNEL_ANSWER`, `CHANNEL_BRIDGE`, `CHANNEL_UNBRIDGE`, `CHANNEL_HANGUP_COMPLETE`, `RECORD_START`, `RECORD_STOP`, `CUSTOM conference::maintenance`, `CUSTOM avmd::beep`, `BACKGROUND_JOB`.

**Reconciliation on reconnect:**
- On connect: `show channels as json` → for every active channel, ensure `call_log` row exists; fetch any missing fields. Emit a `vici2_esl_reconnected_total` metric and a `vici2_esl_reconciled_calls` gauge.

**Pseudocode — Originate:**
```go
func (c *Client) Originate(ctx context.Context, o OriginateOpts) (string, error) {
    uuid := uuidNew()
    vars := buildChannelVars(map[string]string{
        "origination_uuid": uuid,
        "origination_caller_id_number": o.CallerID,
        "campaign_id": o.CampaignID,
        "lead_id":     fmt.Sprint(o.LeadID),
        "tenant_id":   fmt.Sprint(o.TenantID),
        "call_timeout": fmt.Sprint(o.TimeoutSec),
        "hangup_after_bridge": "true",
        "ignore_early_media":  "true",
        "execute_on_answer":   o.OnAnswer, // e.g. "park" or "transfer:conf_42 XML default"
    })
    target := fmt.Sprintf("sofia/gateway/%s/%s", o.CarrierName, o.PhoneE164)
    cmd := fmt.Sprintf("originate %s%s &park()", vars, target)
    jobID, err := c.BgAPI(ctx, "originate", cmd)
    if err != nil { return "", err }
    // optionally wait for BACKGROUND_JOB result
    return uuid, nil
}
```

**Pseudocode — Node listener event flow:**
```
ESL frame → parser → typed event
  ↓
  is critical (HANGUP_COMPLETE, RECORD_STOP)? → DB write (await)
  ↓
  publish to Redis stream events:vici2.call.<event>   (XADD MAXLEN ~ 1M)
  ↓
  if has agent_id, publish to t:{tid}:broadcast:agent:{aid}
  if has campaign_id, publish to t:{tid}:broadcast:campaign:{cid}
  ↓
  metric counter increment
```

## Verification phase
1. Start FS, start dialer, start api. Both ESL connections show RUNNING in `fs_cli > show clients`.
2. Run `node scripts/originate-park.js +12125551212` → call appears in FS, hangup after timeout, all events flow to Redis stream `events:vici2.call.ended`.
3. Kill FS for 30s. Both ESL clients log reconnect attempts with backoff. After FS restarts, both reconnect within 5s.
4. With 2 active calls when FS restarts: after reconnect, `call_log` reflects "ended with cause=ESL_RECONCILED" if FS dropped them, OR rows show in_progress correctly if FS preserved them.
5. Backpressure: synthetic burst of 1000 events → API stays responsive (Redis stream length grows; no event drop).
6. ESL command serialization: 100 concurrent `Originate` calls all complete; no garbled responses.
7. `vici2_esl_*` metrics visible at `:9101/metrics` (api) and `:9102/metrics` (dialer).

## Acceptance criteria
- [ ] Both clients reconnect with exponential backoff (max 30s).
- [ ] Reconcile-on-reconnect populates missed call_log rows.
- [ ] Concurrent commands from many goroutines/promises don't garble.
- [ ] Backpressure: 10x burst handled without event loss.
- [ ] BACKGROUND_JOB callback registry works (originator gets the result).
- [ ] All listed event handlers implemented + tested.
- [ ] No raw ESL strings in caller code (must use typed wrappers).
- [ ] Metrics emitted for connect/disconnect/reconnect/cmd_count/event_count by name.

## Risks
- **Single ESL connection bottleneck.** If 1k+ commands/sec, may need a pool. Phase 1 single connection is fine.
- **Event-loop overload (Node).** If listener handler throws synchronously, listener dies. Wrap every handler in try/catch; emit error metric.
- **State drift between FS and our DB.** Reconcile is the safety net; budget time for it.
