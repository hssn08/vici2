# Module E05 — Drop-Rate Enforcement + Safe-Harbor Message

| Field | Value |
|---|---|
| Track | Dialer Engine + Compliance |
| Phase | 2 |
| Effort | 3–4 days |
| Owner agent type | backend-go + telephony |
| Status | NOT_STARTED |

## Goal
Track abandonment per campaign per 30-day rolling window. When live customer answers but no agent picks within 2s → play safe-harbor audio (TCPA-compliant), write `drop_log`, increment counter. If 30-day drop% > campaign target → throttle E02 to PROGRESSIVE.

## Dependencies
- E02, E04, F04 (drop_window stream), F02 (drop_log, campaigns)

## Blocks
- (none new)

## Public interface
- Function called from answer handler when picker returns ok=false:
  - `RecordDrop(ctx, callUUID, campaignID, reason)` → plays safe-harbor + hangup + writes log.
- `IsDropGated(campaignID) bool` for E02.

## Verification phase
1. Originate to a customer; no agent assigned; safe-harbor audio plays; hangup; drop_log row written.
2. drop_window stream incremented.
3. Drop% > target → gate flips on; E02 throttles within 30s.
4. Drop% drops below target → gate releases.
5. Safe-harbor audio configurable per campaign.

## Test phase
**Unit:** drop% calc with various windows.
**Integration:** end-to-end with SIPp simulating answered call + no agent.

## Acceptance criteria
- [ ] Safe-harbor audio plays before hangup.
- [ ] drop_log + drop_window populated atomically.
- [ ] 30-day rolling drop% accurate.
- [ ] Gate triggers + releases.
- [ ] Per-campaign target.
- [ ] Metric: drop_pct gauge per campaign.
