# Module R02 — Recording Metadata + S3 Upload Worker

| Field | Value |
|---|---|
| Track | Recording |
| Phase | 1 |
| Effort | 4–5 days |
| Owner agent type | backend-node + sre |
| Status | NOT_STARTED |

## Goal
On RECORD_STOP, write `recording_log` row, then encode WAV→MP3 (smaller, browser-friendly), upload to S3 (or local in dev), update row with storage URL, delete local file.

## Dependencies
- R01 (RECORD_STOP event + path)
- F02 (`recording_log`)
- F04 (job queue)

## Blocks (downstream)
- R03 (playback)
- N07 (transcription)

## Public interface
**Worker job:** `vici2:queue:recording-encode`
**Side effects:** `recording_log` row created; `storage_url` populated; local file deleted.

## Research phase
**MCPs:**
- `mcp__exa__web_search_exa`: "Node ffmpeg WAV to MP3 stream production fluent-ffmpeg"
- `mcp__exa__web_search_exa`: "AWS S3 multipart upload Node @aws-sdk/client-s3 large file streaming"
- `mcp__exa__web_search_exa`: "S3 object lock compliance retention 4 years"

**Questions:**
- Encoder: ffmpeg via `fluent-ffmpeg`, or `lame`? **Recommendation:** ffmpeg (most flexible).
- Bitrate: 32 kbps mono is fine for speech; 64 kbps stereo for our 2-channel.
- S3 storage class: Intelligent-Tiering for first 30d, Glacier Instant for older.
- Object lock for compliance: enable bucket-level Compliance mode with 4-year retention.

## Verification phase
1. End a recorded call; within 60s `recording_log` row exists with `storage_url`.
2. MP3 plays correctly in browser; stereo channels intact.
3. Local WAV deleted after upload.
4. S3 object has Object Lock retention set.
5. Job retries on transient S3 failure; dead-letter on permanent.

## Test phase
**Unit:** encode params; URL generation.
**Integration:** real ffmpeg + LocalStack S3.
**Run:** `cd workers && npm test -- recording-encode`

## Acceptance criteria
- [ ] WAV→MP3 encoding works; bitrate as specified.
- [ ] S3 upload + Object Lock applied.
- [ ] Local cleanup after upload confirmed.
- [ ] recording_log populated with all fields (size, duration, URL).
- [ ] Retry on transient failure; dead-letter on permanent.
- [ ] Worker handles 100 concurrent recordings without OOM.

## Risks
- **ffmpeg availability** — bake into worker Docker image.
- **S3 cost** — at 50 agents × 4h/day × 30d × 64kbps stereo = ~280 GB/mo. ~$6/mo at IA tier. Negligible.
