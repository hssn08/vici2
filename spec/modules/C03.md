## Goal
A central audit log table + service called from every privileged action (login, dispo, DNC change, transfer, recording access, etc.). Daily archival to S3 with Object Lock for 4-year retention. App layer forbids UPDATE/DELETE on `audit_log` (DB grants enforce).

## Dependencies

## Blocks

## Public interface
```typescript
class AuditService {
  log(input: AuditEntry): Promise<void>          // INSERT only
  query(filters): Promise<Paginated<AuditEntry>> // admin-only
}
type AuditEntry = {
  tenant_id: number
  event: string                  // 'auth.login', 'dispo.set', 'dnc.add', ...
  actor_user_id?: number
  actor_ip?: string
  resource?: { kind, id }
  payload?: Record<string, unknown>  // details (no PII bulk dumps)
  ts?: Date
}
```

**DB grants:**
- App user has `INSERT, SELECT` on `audit_log`. **No** `UPDATE, DELETE`.
- Admin user (separate) has `DELETE` for retention purges (C04).

3. Daily archival worker writes prior day's rows to S3 with Object Lock.
4. Query API filters by event/actor/resource/date.

## Acceptance criteria
- [ ] App can only INSERT/SELECT.
- [ ] Daily S3 archival with Object Lock.
- [ ] Query API for admin compliance audit.
- [ ] Standard event taxonomy (registry of names).
- [ ] Payload size limit (4KB).

## Risks
- **High-volume events** drown the table — per-event sampling for low-value events (e.g., `agent.heartbeat`).
- **Grant misconfig** in dev — covered by CI integration test.
