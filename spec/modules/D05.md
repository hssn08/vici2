**Pseudocode — isDnc (hot path):**
```typescript
async function isDnc(phone, opts) {
  // L1: Redis (per-source SET membership)
  const sources = [];
  if (opts.useInternal && await redis.sismember(`t:${tid}:dnc:internal`, phone)) sources.push('internal');
  if (opts.useFederal && await redis.sismember(`dnc:federal`, phone)) sources.push('federal');
  if (opts.useState && state && await redis.sismember(`dnc:state:${state}`, phone)) sources.push('state');
  if (opts.useLitigator && await redis.sismember(`dnc:litigator`, phone)) sources.push('litigator');
  return { dnc: sources.length > 0, sources };
}
```

## Risks
- **Real federal sync requires FTC subscription account.** Phase 1 ships with stub; production must purchase SAN ($0–$24k/year depending on area-code count).
- **Cache memory at full federal load** — federal DNC has ~250M numbers; cannot fit in Redis as plain SET. **Mitigation:** use Redis Bloom filter (`BF.RESERVE` from RedisBloom module) for federal+state; fall back to MySQL on positive hit. Plan to enable RedisBloom in F03's module-loading.
