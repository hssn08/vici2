## Goal
Wire SIP.js into the agent web app: connect WSS to FreeSWITCH on login, register, place an outbound call to `*9${user_id}` to land in the agent's conference, auto-answer inbound INVITEs (the dialer pushes calls in), expose hold/mute/DTMF/hangup primitives.

## Dependencies
- F03 (WSS), F05 (sip_creds in login response), T03 (`*9${user_id}` extension), A01 (app shell)

## Blocks (downstream)
- A04, A05, A06, A07, A08, A09 (everything that needs audio control)
- S02 (supervisor eavesdrop reuses SIP.js)

## Public interface
**React hook:** `useSoftphone()` returns `{ status, error, mute, unmute, hold, unhold, sendDtmf, hangup }`
**Component:** `<SoftphoneProvider>` wraps app; instantiates SIP.js once on login.
**Audio element:** hidden `<audio id="remoteAudio" autoplay>` attached to provider.

**Pseudocode:**
```typescript
// SoftphoneProvider
useEffect(() => {
  if (!authStore.sipCreds) return;
  const su = new SimpleUser(authStore.sipCreds.ws_uri, {
    aor: `sip:${authStore.user.id}@${authStore.sipCreds.domain}`,
    userAgentOptions: {
      authorizationUsername: String(authStore.user.id),
      authorizationPassword: authStore.sipCreds.password,
    },
    media: { remote: { audio: audioRef.current } },
    delegate: {
      onCallReceived: () => su.answer(),       // dialer pushes; we answer
      onCallHangup: () => setStatus('idle'),
    },
  });
  await su.connect();
  await su.register();
  await su.call(`sip:*9${authStore.user.id}@${authStore.sipCreds.domain}`); // join own conf
  setSu(su);
  return () => { su.disconnect(); };
}, [authStore.sipCreds]);
```

## Verification phase
1. Log in as agent; mic permission prompt; grant; status=READY.
2. `fs_cli > conference list` shows agent in `conference_${id}`.
3. Make a test call from another browser; bidirectional audio.
4. DTMF: press 5; remote side receives.
5. Hold: remote hears MoH; unhold; audio resumes.
6. Mute: remote silence; unmute; audio resumes.
7. Hangup: leg drops; agent stays in own conf.
8. Network blip (toggle wifi); SIP.js reconnects; audio resumes within 5s.
9. Mic denied: clear UI message; "Try Again" button.
10. Tested on Chrome 120+, Edge, Firefox, Safari.

## Risks
- **Browser audio autoplay policies** — must trigger `audio.play()` in a user-gesture context after login.
- **Safari WebRTC quirks** — test specifically.
- **Codec mismatch** — FS internal profile must include OPUS + PCMU; browser sends OPUS preferred; SDP negotiation handled by SIP.js.
