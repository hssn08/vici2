# Module S01 — Live Wallboard

| Field | Value |
|---|---|
| Track | Supervisor |
| Phase | 3 |
| Effort | 4–5 days |
| Owner agent type | frontend + backend-node |
| Status | NOT_STARTED |

## Goal
Real-time supervisor dashboard: agent grid (status, on-call timer, current campaign/lead), per-campaign metrics (ready/inCall/paused, hopper depth, drop% gauge, calls/sec), per-ingroup queue depth + longest wait.

## Dependencies
- A03 (WS), F04 (Redis), M01

## Public interface
**Page:** `(sup)/wallboard`
**WS subscription:** `t:{tid}:broadcast:wallboard`

## Plan phase
**Deliverable:** `spec/modules/S01/PLAN.md`: layout; metric set; refresh cadence; large-screen mode.

## Implementation phase

## Verification phase
1. With 20 active agents + 2 campaigns + 1 ingroup, all panels render with live data.
2. Agent state changes reflect within 1s.
3. Drop gauge color-codes (green < 1.5%, amber < 3%, red ≥ 3%).
4. Large-screen / TV mode (full-screen toggle).

## Test phase
**Playwright** with seeded state.

## Acceptance criteria
- [ ] All panels live.
- [ ] < 1s latency on state changes.
- [ ] Color-coded drop gauge.
- [ ] TV / kiosk mode.
- [ ] Per-campaign filter.

## Risks
- **Information overload** — let supervisor pick which campaigns/ingroups to watch.
