## Goal
Deploy Kamailio in front of multiple FS instances. SIP signaling enters Kamailio (5060), dispatcher module load-balances to FS pool with health probes + failover.

## Dependencies
- F03, T03

## Public interface
- Kamailio listening on 5060 (UDP/TCP/TLS) and 7443 (WSS).
- Dispatcher set #1: outbound to FS pool. Set #2: inbound to FS pool.
- Health probes (OPTIONS) every 15s; mark inactive after 3 misses.

## Verification phase
1. 2 FS instances behind Kamailio; new SIP REGISTERs balanced.
2. Kill one FS; Kamailio detects within 45s (3 × 15s); routes to remaining.
3. Bring FS back; rejoins pool.
4. Call-id affinity: re-INVITEs/BYEs reach same FS.

## Acceptance criteria
- [ ] Round-robin or fewest-active dispatch.
- [ ] OPTIONS health probes.
- [ ] Failover within 45s.
- [ ] Affinity by call-id.
- [ ] Documented HA setup (VRRP/keepalived).
