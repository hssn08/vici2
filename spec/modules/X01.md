## Goal
Co-locate `rtpengine` with FS to offload SRTP↔RTP transcoding. Solves the documented ~150 WebRTC concurrent ceiling on FS-only deployments.

## Dependencies
- F03 (FS); ideally before agent count crosses 100.

## Public interface
- rtpengine listening on local UDP port 22222 (control) + RTP range
- FS configured to use rtpengine via `mod_rayo`-style hook OR `nathelper` integration

## Verification phase
1. WebRTC call: SRTP terminates at rtpengine; FS sees plain RTP.
2. Load test: 200 concurrent WebRTC sessions stable; no packet loss.
3. CPU on FS halved; rtpengine takes the SRTP load.
