## Goal
Maintain monthly partitions on `call_log` / `agent_log` / `recording_log`; archive partitions older than 4 years to S3 then drop them; delete S3 objects past retention.

## Dependencies
- F02 (partitioned tables), C03 (S3 setup)

## Public interface
**Worker runs daily at 03:00:**
1. Ensure next 3 months of partitions exist (CREATE PARTITION).
2. For each table: archive partitions > 4 years old to S3, then `ALTER TABLE ... DROP PARTITION`.
3. S3 lifecycle policy auto-deletes archived data > 5 years (one year buffer).
4. Updates `data_retention_summary` table for admin visibility.

## Verification phase
1. Run with seeded data including a partition > 4y old; partition archived + dropped.
2. New partition for next month created.
3. `data_retention_summary` updated.
4. Restart worker mid-run → resumes safely.

## Test phase
**Integration** with seeded test data and LocalStack S3.

## Acceptance criteria
- [ ] Future partitions created proactively.
- [ ] Old partitions archived + dropped.
- [ ] S3 lifecycle policy confirmed at one year buffer past mandatory retention.
- [ ] Idempotent on retry.
- [ ] Audit log per archival run.

## Risks
- **Accidental drop of in-use partition** — guard with check that partition's max date < cutoff.
- **S3 archival failure** must NOT result in partition drop — order: archive-then-drop, with verification.
