# Module E06 — Channel + Conference Janitor

| Field | Value |
|---|---|
| Track | Dialer Engine + Operations |
| Phase | 1 (start early) / 2 (auto-dial pressure increases need) |
| Effort | 2–3 days |
| Owner agent type | backend-go |
| Status | NOT_STARTED |

## Goal
Periodic sweep that finds stuck channels (call_log started > 4h ago without ended), stale conferences (no members for > 5 min that aren't an agent's home conf), orphaned hopper locks. Vicidial learned this the hard way; we don't.

## Dependencies
- T01 (ESL), F02 (call_log)

## Blocks
- (none — defensive)

## Public interface
- Background loop every 60s
- Acquires Redis lock so only one instance sweeps

## Plan phase
**Deliverable:** `spec/modules/E06/PLAN.md`: sweep queries; safety thresholds; alerts.

## Implementation phase
```
dialer/internal/janitor/janitor.go

```

## Verification phase
1. Insert a "stuck" call_log row; sweep marks it ENDED with cause=JANITOR.
2. Create an empty conference; sweep `conference list`; deletes after 5 min.
3. Orphaned hopper lock cleared.
4. Lock prevents double-sweep on multi-instance.

## Test phase
**Unit:** sweep predicates.
**Integration:** real FS + DB.

## Acceptance criteria
- [ ] All three sweep types implemented.
- [ ] Single-instance via Redis lock.
- [ ] Metrics + alerts on sweep activity (high counts = bug elsewhere).
- [ ] Safe thresholds (won't kill in-flight calls).

## Risks
- **Killing live calls due to clock drift** — use FS-reported timestamps where possible.
