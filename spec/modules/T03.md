## Goal
The "agent always sits in their own conference" primitive. When agent SIP.js calls `*9${user_id}`, FreeSWITCH answers and joins them to `conference_${user_id}@default`. Provides agent-presence eventing back through ESL so the API knows who's logged in.

## Dependencies

- F05 (per-user SIP creds + directory entries)
- T01 (ESL events for conference::maintenance)

## Blocks (downstream)
- A02 (browser softphone targets `*9${user_id}`)

- I04 (closer/blended uses agent conf)
- S02 (eavesdrop)

## Public interface
**FS dialplan extension(s):**
- `*9NNNN` — agent join (NNNN = user_id; verified against caller's SIP user)
- `*8NNNN` — agent listen (used for cross-conf supervision)
- `conference_${user_id}@default` — naming convention; never used outside dialplan or ESL

**Channel variables set on agent leg:**
- `vici2_user_id`, `vici2_tenant_id`, `vici2_role=agent_legs`
- `vici2_conf_name=conference_${user_id}`

**Events emitted:**
- `CUSTOM conference::maintenance member-add` with the agent join → API consumer flips agent state to `READY` (after auth check).

- Should we use `conference $name+flags{moderator,mute=false,nomoh}`? `nomoh` flag avoids MOH on single-member conf.

**Pseudocode — dialplan extension:**
```xml
<extension name="agent_conference_join">
  <condition field="destination_number" expression="^\*9(\d+)$">
    <!-- verify caller matches the requested user_id -->
    <condition field="${sip_from_user}" expression="^${1}$">
      <action application="answer"/>
      <action application="set" data="vici2_user_id=$1"/>
      <action application="set" data="vici2_role=agent_leg"/>
      <action application="set" data="vici2_conf_name=conference_$1"/>
      <action application="set" data="conference_auto_record=false"/>
      <action application="conference" data="conference_$1@default+flags{nomoh}"/>
      <anti-action application="hangup" data="USER_NOT_REGISTERED"/>
    </condition>
    <anti-action application="hangup" data="UNAUTHORIZED"/>
  </condition>
</extension>
```

**Pseudocode — agent presence handler:**
```typescript
// when CUSTOM conference::maintenance arrives
on('member-add', evt => {
  const userId = evt.getHeader('vici2_user_id');
  if (!userId || !evt.getHeader('Conference-Name').endsWith(userId)) return;
  await redis.agent.setStatus(tenantId, userId, 'READY', { server, conf: evt.getHeader('Conference-Name') });
  await audit({ event: 'agent.online', user_id: userId });
});
on('member-remove', evt => {
  // only flip to OFFLINE if no other member-add in 5s (handles re-join blips)
  await debounceUserOffline(userId, 5000);
});
```

## Verification phase
1. Agent SIP.js (mock or real) calls `sip:*91042@fs.example.com` with auth user `1042`. Audio answers; FS shows agent in `conference_1042` via `conference list`.
2. API agent state for user 1042 flips to READY in Redis within 1s.
3. SIP user 1042 calls `*91099` (someone else's conf) → call rejected with UNAUTHORIZED.
4. Agent disconnects (browser closed) → after 5s grace, status → OFFLINE in Redis.
5. Agent reconnects within 5s → no flicker; remains READY.
6. 50 concurrent agent joins → all join their own conferences; CPU ≪ 5%.
7. Empty conferences sit idle without MOH (verify with `conference list` showing 1 member, no moh playing).

## Acceptance criteria
- [ ] Agent join only succeeds when SIP user matches requested user_id.
- [ ] Agent presence flips to READY within 1s; OFFLINE within 5s of disconnect.
- [ ] No MOH on idle conferences.
- [ ] 100 concurrent agent confs sit idle at <5% CPU on reference hardware.
- [ ] Re-join within grace window doesn't flicker state.
- [ ] Audit logs emitted for agent.online/offline.
- [ ] Cross-conf join attempt rejected.

## Risks
- **`${sip_from_user}` vs `${sip_authorized_user}`** — pick the post-auth one to prevent spoofing. Test both.
- **Grace-window debounce** — too short = flicker, too long = stale READY. 5s is a reasonable default; tune per ops feedback.
