**Pseudocode for partitioned table migration:**
```sql
-- 003_init_logs.sql (raw, not Prisma-managed model)
CREATE TABLE call_log (
  id            BIGINT NOT NULL AUTO_INCREMENT,
  tenant_id     BIGINT NOT NULL DEFAULT 1,
  uuid          VARCHAR(40) NOT NULL,
  -- ... per DESIGN.md §5
  call_started  DATETIME NOT NULL,
  PRIMARY KEY (id, call_started),                 -- partition col must be in PK
  UNIQUE KEY uk_call_log_uuid (uuid, call_started),
  KEY idx_campaign_started (campaign_id, call_started),
  KEY idx_lead (lead_id),
  KEY idx_started (call_started)
)
PARTITION BY RANGE (TO_DAYS(call_started)) (
  PARTITION p_pre VALUES LESS THAN (TO_DAYS('2026-01-01')),
  PARTITION p_2026_05 VALUES LESS THAN (TO_DAYS('2026-06-01')),
  PARTITION p_2026_06 VALUES LESS THAN (TO_DAYS('2026-07-01')),
  -- C04 worker maintains future partitions monthly
  PARTITION p_max VALUES LESS THAN MAXVALUE
);
```

**Prisma model annotation for the same:**
```prisma
/// @ignore for FK; managed by raw migrations
model CallLog {
  id           BigInt   @id @default(autoincrement())
  tenantId     BigInt   @default(1) @map("tenant_id")
  uuid         String   @db.VarChar(40)
  callStarted  DateTime @map("call_started")
  // ... all fields
  @@map("call_log")
  @@index([campaignId, callStarted], map: "idx_campaign_started")
  @@index([leadId], map: "idx_lead")
}
```

## Verification phase
**Manual checks:**
1. `npx prisma migrate dev` from empty DB completes; all tables present.
2. `npx prisma db seed` succeeds; `SELECT count(*) FROM phone_codes` returns 800+.
3. `SELECT count(*) FROM statuses` returns 10+.
4. Insert a fake `call_log` row dated 2026-12-15 → goes into a partition (verify via `EXPLAIN PARTITIONS SELECT * FROM call_log WHERE call_started='2026-12-15'`).
5. `npx prisma migrate reset` → wipes and re-runs cleanly.
6. Generated Prisma client compiles in `api/` and `workers/`.
7. ERD doc (mermaid) renders without errors in `docs/db/erd.md`.
