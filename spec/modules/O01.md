## Goal
Prometheus scrapes every service's `/metrics` endpoint; Grafana dashboards for: dialer (originates/sec, drop%, hopper depth, pacing accuracy), agents (state breakdown, productivity), FS (calls, conferences, RTP, ESL), MySQL, Redis, hosts. Alerts via Alertmanager.

## Dependencies
- F01 (services exposing /metrics)

## Public interface
- `prom/prometheus.yml` scrape config
- `grafana/dashboards/*.json` provisioned dashboards
- Alertmanager rules (drop% > 2.5%, FS down, MySQL replication lag, ESL disconnect)

## Plan phase
**Deliverable:** `spec/modules/O01/PLAN.md`: scrape topology; dashboard set; alert thresholds; on-call escalation.

## Implementation phase
```
docker-compose.observability.yml             # Prom + Grafana + Alertmanager
prom/prometheus.yml
prom/rules/*.yml
grafana/dashboards/{dialer,agents,freeswitch,mysql,redis,hosts}.json
grafana/datasources/prometheus.yml
spec/runbooks/oncall.md
```

## Verification phase
1. All services scraped within 60s.
2. Dashboards populate.
3. Force a drop% violation in test → alert fires within 60s.
4. FS down → alert fires within 60s.

## Acceptance criteria
- [ ] All services instrumented.
- [ ] 6+ dashboards.
- [ ] Critical alerts defined.
- [ ] On-call runbook.
- [ ] Lighthouse-grade Grafana load times.
