## Goal
Per-campaign tick loop (every `1000ms / calls_per_second`) that computes `desired = ready_agents × dial_level - active_calls` and originates that many leads from the hopper via T04. Coordinates across multiple dialer instances via Redis lock so each tick fires once.

## Dependencies
- E01 (hopper), T04 (originate), F04 (Redis), F02 (campaigns)

## Blocks
- E03 (adaptive feeds dial_level), E05 (drop-rate enforcement throttles)

## Public interface
- `Engine.startCampaign(cid)` / `Engine.stopCampaign(cid)`
- Reads `t:{tid}:campaign:{cid}:dial_level` (E03 writes)
- Writes `vici2_dialer_originate_total` metric

**Pseudocode:**
```go
func (p *Pacer) tick(ctx context.Context, cid string) error {
    locked, err := p.redis.AcquireTickLock(cid, 800*time.Millisecond)
    if err != nil || !locked { return nil }
    c := p.cfg.Get(cid)
    if c.DialMethod == "MANUAL" || !c.Active { return nil }
    level := p.levelFor(c)            // RATIO: c.AutoDialLevel; ADAPT_*: redis.Get(dial_level)
    agents := p.agentCount(cid, c.AvailableOnlyTally)
    active := p.redis.SCard(activeCallsKey(cid))
    desired := int(math.Round(float64(agents)*level)) - active
    if desired <= 0 { return nil }
    if e5.IsDropGated(cid) { desired = min(desired, 1) }
    // pop lead and originate
    for i := 0; i < desired; i++ {
        leadID, ok := p.redis.PopHopperWithLock(cid)
        if !ok { break }
        go p.originateOne(c, leadID)
    }
    return nil
}
```

## Verification phase
1. Campaign RATIO=2.0 with 5 ready agents → ~10 active outbound.
2. PROGRESSIVE with 5 agents → 5 active outbound.
3. Switch to ADAPT_TAPERED with reading from dial_level key → matches that value.
4. Stopping campaign halts all new originates within 1s.
5. Two dialer instances running → no double-tick (Redis lock).
6. Drop% gate (E05) reduces pacing when drop > target.

## Test phase
**Unit:** decision math; throttle policies.
**Integration:** real Redis + mocked T04; assert correct number of originate calls.

## Acceptance criteria
- [ ] All 5 dial methods (MANUAL/RATIO/PROGRESSIVE/ADAPT_*) supported.
- [ ] Multi-instance safe via Redis lock.
- [ ] Drop% gate honored.
- [ ] Per-carrier cap honored.
- [ ] Metrics: originates/sec, desired vs actual.
- [ ] No origination after `stopCampaign`.

## Risks
- **Tick drift across instances** — lock dedupes but not if instance clocks differ. Use Redis as time source.
- **Burst on sudden agent ready** — smooth via per-second cap.
