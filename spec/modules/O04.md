## Goal
GitHub Actions (or GitLab CI) pipeline: lint → unit tests → build → integration tests → docker images → staging deploy on merge to main, prod deploy on tag.

## Dependencies
- F01

## Public interface
- `.github/workflows/{ci,deploy-staging,deploy-prod}.yml`
- Container registry (GHCR or ECR)

## Plan phase
**Deliverable:** `spec/modules/O04/PLAN.md`: workflow stages; secrets management; environments; rollback strategy.

## Implementation phase
```
.github/workflows/ci.yml
.github/workflows/deploy-staging.yml
.github/workflows/deploy-prod.yml
.github/workflows/load-nightly.yml
scripts/deploy/staging.sh
scripts/deploy/prod.sh
scripts/deploy/rollback.sh
```

## Verification phase
1. Push to PR → CI passes.
2. Merge to main → staging deploy.
3. Tag v0.1.0 → prod deploy with manual approval.
4. Rollback script reverts to previous version.

## Acceptance criteria
- [ ] All stages pass.
- [ ] Secrets in env (never in code).
- [ ] Staging + prod environments.
- [ ] Rollback works.
- [ ] PR template enforces module-id reference.

## Risks
- **Database migrations on prod deploy** — run separately, with admin approval.
