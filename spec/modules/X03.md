# Module X03 — Multi-FS Campaign Affinity

| Field | Value |
|---|---|
| Track | Scale-out |
| Phase | 3.5 |
| Effort | 5–7 days |
| Owner agent type | backend-go + sre |
| Status | NOT_STARTED |

## Goal
Pin each campaign's agents + originates to one FS instance to keep all conferences for a campaign on the same box (avoids cross-FS audio bridging). Dialer engine picks the right ESL host per campaign.

## Dependencies
- X02, E02

## Public interface
- `campaigns.fs_node_id` FK → `fs_nodes.id` (NULL = auto-assign pending).
- Dialer maintains `map[fs_node_id]*esl.Conn` via `dialer/internal/esl/router.go`; routes originates accordingly.
- Re-balancer worker re-pins campaigns when FS pool changes.
- RBAC verbs: `infra:fs_node:read`, `infra:fs_node:edit`.
