## Goal
Admin UI for users and user_groups: create/edit/disable/delete; assign roles; rotate SIP password; assign user groups; assign campaigns/in-groups per group.

## Dependencies
- F05, M01

## Public interface
**REST:** `/api/admin/users`, `/api/admin/user-groups`.
**UI pages:** `(admin)/users`, `(admin)/user-groups`.

## Plan phase
**Deliverable:** `spec/modules/M05/PLAN.md`: user form, group form, password reset flow.

## Verification phase
1. Create agent user; SIP creds generated; FS XML written; user can log in + register.
2. Disable user; can't log in; existing session terminated.
3. Rotate SIP password; old password fails next register.
4. Create user_group; assign campaigns; users in group inherit access.
5. Delete user (soft); record kept for audit.

## Test phase
**Playwright** + unit tests on services.

## Acceptance criteria
- [ ] CRUD for users + groups.
- [ ] SIP password rotation.
- [ ] Disabled-user flow (sessions terminated).
- [ ] Group → campaign access matrix enforced in agent UI.
- [ ] Audit log per change.

## Risks
- **Active session termination on disable** — must publish to that user's WS to force logout.
